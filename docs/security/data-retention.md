# データ保持ポリシー

最終更新日: 2026-05-27

このドキュメントは、Lunariaで扱うデータの保持期間、削除方針、アクセス制御の考え方を整理します。

## 基本方針

- 必要な期間だけ保存する
- ギルド単位でデータを分離する
- 削除申請に対応できる設計にする
- 重要操作の監査ログは改ざんしにくい形で保存する
- RecordingやAI処理結果はRestricted領域として扱う

## データ別方針

| データ | 保持方針 | 備考 |
|---|---|---|
| Guild設定 | 利用中は保持 | 解除時に削除対象 |
| PluginSetting | 利用中は保持 | Config History追加後は履歴管理 |
| Rule | 利用中は保持 | 無効化と削除を分ける |
| AuditLog | 原則1年 | プランにより延長余地あり |
| Quote | ギルド管理者が非表示・削除可能 | 削除より非表示を基本にする |
| GameAccount | 連携解除まで | UIDやゲームIDは慎重に扱う |
| Recording | 最大180日を上限候補 | 実装前に再確認 |
| Transcription | Recordingと同じ方針 | 削除導線必須 |
| APIログ | 短期保持 | 認証情報は含めない |

## 削除導線

将来的に以下を提供します。

- ユーザー連携解除
- ゲームID削除
- Quote非表示・削除
- 録音データ削除申請
- ギルドデータ削除申請

## 注意

このドキュメントは初期方針です。Public Alpha前に利用規約、プライバシーポリシー、録音ポリシーと整合させます。
